# منصة خطة محتوى المدرسين (Django)

## طريقة التشغيل

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py createsuperuser   # اختياري - لوحة الأدمن
python manage.py runserver
```

افتح المتصفح على: http://127.0.0.1:8000/

## رحلة المستخدم
1. **إنشاء حساب** على `/signup/`
2. بعد التسجيل مباشرة، هيتوجه لشاشة **إضافة المعلم** `/add-teacher/` (مرة واحدة فقط لكل حساب) - يكتب اسم المستر.
3. بمجرد الحفظ، هيتم إنشاء **لوحة التحكم** الخاصة به على `/dashboard/` معبأة بنفس خطة المحتوى الافتراضية (32 فكرة) القابلة للتعديل بالكامل.
4. تسجيل الدخول مرة تانية (`/login/`) هيوديه على الداشبورد بتاعته على طول (مفيهوش شاشة إضافة مستر تاني لأنه أصلاً عنده واحد).

## العزل بين اليوزرز
كل يوزر مرتبط بـ `Teacher` واحد (One-To-One)، وكل عناصر المحتوى (`ContentItem`) مرتبطة بهذا المعلم فقط.
كل الـ API endpoints بتفلتر تلقائيًا على `request.user.teacher` فمستخدم معينش يقدر يشوف أو يعدل بيانات مستخدم تاني.

## الإعدادات قبل النشر (Production)
- غيّر `SECRET_KEY` في `config/settings.py`
- اجعل `DEBUG = False`
- حدد `ALLOWED_HOSTS` بدومينك الحقيقي بدل `['*']`
- استخدم قاعدة بيانات حقيقية (Postgres مثلاً) بدل SQLite لو الترافيك كبير
